### August 2021
* **Payment fund statuses**: Add support for payment fund statuses. ([docs](/docs/api/core/payments/object))

* **Money Transfer Element**: Added a demo for Money Transfer use case. ([docs](/docs/api/elements/money-transfer/intro))

* **Health Check**: Added health check API endpoint `/ping`. ([docs](/docs/api/healthcheck))
* **Webhook IP whitelists**: List possible webhook IP addresses for each Method environment. ([docs](/docs/api/core/webhooks/ip-whitelisting))
* **Idempotency**: Added optional `Idempotency-Key` header for all `POST` Core API resources. ([docs](/docs/api/idempotency))
* **List accounts filters**: Added optional filter parameters for `GET /accounts`. ([docs](/docs/api/core/accounts/list))
* **Generate Reports**: Allow users to generate reports using the `/reports` endpoints. ([docs](/docs/api/core/reports/object))
* **API Client for Node.js**: Created an API client library for node.js. ([docs](../18.%20Libraries/1-Client.md))

### July 2021

* **Guides**: Added a new guides section on the docs. Check out the [first guide](../14.%20IntegrationGuides/1-IntegratingUsingPlaid.md).
* **Elements**: Added a new elements section on the docs. Elements will encapsulate all embeddable UI components, including Link. ([docs](/docs/api/elements/intro))
* **Link deprecation**: Deprecated Method Link. Link will now be a part of Method Elements. ([docs](/docs/api/elements/intro))

### June 2021

* **Entity capabilities**: An entity will have specific capabilities based on the data provided. ([docs](/docs/api/core/entities/object#entity-capabilities))
* **Receive Only entites**: This entity type is being deprecated. See docs for more info. ([docs](/docs/api/core/entities/object#entity-capabilities))
* **Mask in accounts**: A liabilities' mask (last 4) is now returned in the Account object. ([docs](/docs/api/core/accounts/object))

### May 2021

* **Merchant logos**: Extended coverage of logos of merchants ([docs](/docs/api/core/merchants/object))
* **Receive Only entites**: Allows you to push money to an entities' accounts without needing extensive KYC / AML ([docs](/docs/api/core/entities/create-receive-only))
* **Payment simulator**: The dev environment now simulates a payment moving through it's various states. ([docs](/docs/api/environments))
* **Improved Plaid & MX support**: Significantly improved coverage of merchants mapping to Plaid & MX aggregators ([docs](/docs/api/core/merchants/list))
